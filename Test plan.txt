Тест план для проверки корректности операции деления.

Дано:

Графический интерфейс приложения сожержит следующие элементы управления:
- три текстовых поля: делимое А, делитель В и частное С;
- кнопка «=».

При нажатии на кнопку «=» выполняется операция деления, результат выводится в поле С.
Операция выполняется по всем правилам деления. Ввод в поля немаскированный.
При выполнении недопустимых действий пользователю выдается ошибка в поле С по типу:
"Error: <тип ошибки>"

Реализовать:
-Набор тестов для проверки модуля Divider, который реализует операцию деления;
-Набор UI тестов для проверки готового приложения.

Позитивные тесты:
- Целочисленное деление (результат деления - целое число);
- Дробное деление
    a)результат деления в виде конечной дроби;
    b)результат деления в виде бесконечной дроби (точность при сравнении 1e-10);
- Деление со знаком.

Негативные тесты:
- Деление с нулем (A\B\AB);
- Деление с пустыми значениями (A\B\AB);
- Деление с больше, чем одной плавающей точкой;
- Деление с некорректными символами ( любой символ, отличный от цифры).
